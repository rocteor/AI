# 硬件产品案例拆解与踩坑实录

> 从真实产品反推设计方案，理解每个决策背后的成本、风险和 trade-off。
> 比读一百篇教程更有用的是：看别人怎么做出来、踩过什么坑。

---

## 前言：怎么读这份文档

每个案例按统一结构拆解：

1. **产品定位** —— 卖给谁、卖多少钱、核心竞争力
2. **方案选型** —— 为什么选这个芯片/传感器/通信方案
3. **BOM 拆解** —— 每个零件多少钱，整机成本结构
4. **设计决策** —— 几个关键 choices 和 trade-off
5. **踩坑复盘** —— 实际项目中遇到过的问题
6. **PM 要记住的** —— 这个案例最值得带走的一课

案例从简到繁排列，序号不代表难度排名。

## 案例一：智能插座

### 产品定位

- 产品：WiFi 智能插座，手机远程开关 + 定时 + 电量统计
- 目标售价：39-59 元
- 目标用户：家庭用户，希望用手机控制家电
- 竞品：小米智能插座、BroadLink

### 方案选型

**两代方案对比：**

| 维度 | 第一代（2018）| 第二代（2022）|
|------|-------------|-------------|
| MCU | STM32F030F4P6 + ESP8266（双芯片）| ESP32-C3（单芯片，内置 WiFi/BLE）|
| 电量计量 | HLW8032（独立计量芯片）| HLW8032（无法集成）|
| 继电器 | HF32F-005-1HS | HF32F-005-1HS |
| 电源 | 阻容降压（非隔离）| 非隔离 DCDC（MP2451）|
| 成本 | ~25 元 | ~18 元 |

**为什么第一代用双芯片？**
2018 年 ESP8266 的 GPIO 太少、Flash 太小，跑不了复杂的业务逻辑。所以用 STM32 做主控，ESP8266 只当 WiFi 透传模块用。2022 年 ESP32-C3 的 Flash 和 RAM 都足够，单芯片就够了。

**这就是 PM 要关注的趋势：** 芯片集成度越来越高，芯片数量在减少，BOM 在降低，但一次换方案的验证周期要 2-3 个月。

### BOM 拆解（第二代）

| 部件 | 型号 | 单价（10000pcs）| 备注 |
|------|------|--------------|------|
| 主控 | ESP32-C3 | 8 元 | 国产乐鑫，性价比极高 |
| 电量计量 | HLW8032 | 2 元 | |
| 继电器 5V | HF32F | 2 元 | |
| 阻容降压 | 若干 | 1.5 元 | |
| PCB | 2 层板 30×30mm | 2 元 | |

| 外壳+插脚 | 注塑+铜插脚 | 5 元 | 开模费 ~3 万，均摊后 |
| 包装+配件 | 说明书+盒 | 1.5 元 | |
| **BOM 合计** | | **~23 元** | |
| **出厂价** | | **~35-45 元** | |

**PM 要记住：** BOM 23 元的产品，零售价 49-79 元。差价覆盖研发（~20 万）、认证（~5 万）、开模（~3 万）、渠道、售后、利润。

### 设计决策：隔离 vs 非隔离电源

智能插座需要把 220V 交流电降压到 3.3V/5V 给 MCU 供电。两种方案：

| | 阻容降压（非隔离）| DCDC（非隔离）| 隔离电源模块 |
|------|-------------|-------------|------------|
| 成本 | 1.5 元 | 3 元 | 8 元 |
| 安全性 | ⚠️ 低压部分和 220V 共地 | ⚠️ 低压部分和 220V 共地 | ✅ 完全隔离 |
| 体积 | 小 | 中 | 大 |
| 法规 | 需要通过电气间隙测试 | 需要通过电气间隙测试 | 最安全 |

**现实选择：** 绝大多数 39-79 元的智能插座都用非隔离方案。外壳塑料如果设计合理（没有用户能摸到里面的金属），法规测试可以通过。**隔离电源是好，但 8 元 vs 1.5 元的成本差决定了产品定价的上限。**

### 踩坑复盘

**坑 1：WiFi 配网成功率**
- 问题：用户反馈"插座连不上 WiFi"
- 原因：ESP32 的 SmartConfig 配网在 2.4GHz 信道 1-11 上广播，但用户路由器如果固定在信道 12/13，配网永远失败
- 解决：增加蓝牙辅助配网——先用 BLE 把 WiFi 信息传给设备，再切 WiFi 连网

**PM 学到的东西：** WiFi 配网是 IoT 产品退货率最高的原因之一。配网方案不是技术选型，是用户体验的核心环节。BLE + WiFi 双模配网比纯 WiFi 配网的平均成功率从 70% 提升到 98%。

**坑 2：继电器粘连**
- 问题：插座用了一段时间后，继电器关不了（一直通电）
- 原因：继电器触点在大电流（10A）关断时会产生电弧，电弧烧蚀触点，最终导致触点熔焊
- 解决：加过零检测电路，在交流电过零点关断继电器（此时电流最小，电弧最小）

**PM 学到的东西：** 这个坑在批量生产前很难复现——第一次打样 10 片板测不出来，要跑几百次开关才出现。这就是为什么"加速老化测试"是硬件产品量产前的必要环节，不能省。

## 案例二：智能门锁

### 产品定位

- 产品：指纹/密码/蓝牙三合一智能门锁
- 目标售价：499-799 元（出厂价 ~200-350 元）
- 目标用户：年轻家庭、租房用户
- 核心竞争力：安全 + 续航 + 便捷

### BOM 拆解

| 部件 | 说明 | 单价 |
|------|------|------|
| MCU+蓝牙 | Nordic nRF52832 或国产 TLSR8253 | 15-25 元 |
| 指纹模组 | 电容式指纹传感器 + 算法芯片 | 20-30 元 |
| 电机驱动 | 微型减速电机 + 驱动 MOS | 8-15 元 |
| 机械锁体 | 锁芯+锁舌+弹簧结构 | 40-80 元 |
| 电池 | 4 节 AA 或锂电池包 | 10-20 元 |
| PCB | 2 层或 4 层 | 5-10 元 |
| 外壳 | 锌合金压铸 + 表面处理 | 30-60 元 |
| 其他 | 按键、蜂鸣器、LED、NFC | 5-10 元 |
| **BOM 合计** | | **~130-240 元** |

注意 BOM 里最贵的不是芯片，是**机械锁体**和**外壳**。做硬件产品，机械部分的成本往往被 PM 低估——它决定了整机成本的 40% 以上。

### 设计决策：电池续航

门锁是典型的"电池供电、要求超长续航"产品。面对的约束：

| 约束 | 数值 |
|------|------|
| 电池容量 | 4 节 AA 约 2000mAh（串联 6V）|
| 目标续航 | 6-12 个月 |
| 工作电流 | 电机开锁瞬间 ~500mA |
| 待机电流 | 目标 < 5μA |

**待机电流 < 5μA 是什么概念？**
- MCU 的睡眠模式本身就要 1-2μA
- 蓝牙广播间隔从 100ms 拉到 1000ms，电流从 10μA 降到 2μA
- 指纹模组要"按需唤醒"——平时不供电，检测到手指触摸才上电

**设计取舍：**
- 指纹模组平时断电 → 上电到识别完成要 300ms → 用户体验变慢
- 蓝牙始终广播 → 手机靠近自动解锁体验好 → 但待机电流翻倍
- **选择妥协方案：** 断电指纹，但在外壳外面做一个电容触摸电极，检测到人手触摸时唤醒 MCU + 指纹模组。触摸耗电几乎为 0。

**PM 学到的东西：** 续航、性能、成本——硬件产品永远在这三者之间找平衡。没有"既要又要"。

### 踩坑复盘

**坑 1：指纹识别率冬天骤降**
- 问题：冬天用户反馈"指纹打不开"，夏天正常
- 原因：干冷天气导致手指皮肤干燥，电容式指纹传感器对干手指的识别率大幅下降
- 解决：算法升级，增加干手指的匹配模式；或者在传感器选型阶段预留射频式指纹方案（成本高 3 倍）

**PM 学到的东西：** 环境因素（温度、湿度）是硬件产品特有的测试维度。软件产品的测试环境相对单一，硬件产品要在高低温箱里测过才能上市。

**坑 2：蓝牙休眠后被系统杀死**
- 问题：手机 App 在后台被系统杀掉后，门锁蓝牙无法唤醒
- 原因：iOS 和 Android 对后台蓝牙应用有不同的限制策略
- 解决：Android 用厂商白名单引导用户设置；iOS 用蓝牙后台模式 + iBeacon 区域事件

**PM 学到的东西：** App 端的兼容性问题有时比硬件更头疼。硬件做完 80% 之后，App 端还要花 80% 的时间做兼容性适配。

## 案例三：智能空气净化器

### 产品定位

- 产品：中端智能空气净化器，CADR 值 300m³/h
- 目标售价：999-1499 元
- 目标用户：家庭用户、过敏人群、有小孩的家庭

### 方案选型

```
方案架构：
┌──────────────────────────────────────┐
│              主控 MCU                  │
│         STM32F103RET6                 │
│  ┌─────┬─────┬──────┬─────┬──────┐   │
│  │ PM2.5 │ 温湿度 │ 显示 │ 电机 │ WiFi │
│  │ 传感器 │ 传感器 │ SPI  │ PWM  │ UART │
│  └─────┴─────┴──────┴─────┴──────┘   │
└──────────────────────────────────────┘
```

**为什么不选 ESP32？**
空气净化器的功能比智能插座复杂得多：需要同时驱动 PM2.5 传感器（I2C）、OLED 显示屏（SPI）、直流无刷电机（PWM）、WiFi 通信（UART 外接）。ESP32 虽然集成了 WiFi，但引脚不够用，且它的 ADC 精度不足以做 PM2.5 传感器的校准。

**PM 要知道：** 选芯片不是"功能够用就行"，是要"所有外设的引脚不冲突 + 性能和精度达标"。

### BOM 拆解

| 部件 | 说明 | 单价（1000pcs）|
|------|------|--------------|
| MCU | STM32F103RET6 | 25 元 |
| PM2.5 传感器 | 攀藤 PMS5003（激光散射）| 40 元 |
| 温湿度 | SHT30 | 8 元 |
| OLED 屏幕 | 0.96寸 SSD1306（SPI）| 15 元 |
| 直流无刷电机 | 含驱动板 | 60 元 |
| WiFi 模组 | ESP8266 透传模块 | 12 元 |
| PCB | 4 层板 | 15 元 |
| 电源 | 24V/1A 适配器 | 20 元 |
| 外壳 | 注塑 ABS | 50 元 |
| 滤芯 | HEPA + 活性炭复合 | 30-40 元 |
| 其他 | 按键、LED、风扇、线束 | 20 元 |
| **BOM 合计** | | **~295 元** |
| **出厂价** | | **~450-600 元** |
| **零售价** | | **~999-1499 元** |

### 设计决策：滤芯是耗材

空气净化器最特别的一点：**BOM 里的滤芯是耗材**，用户每隔 3-6 个月就要买一次。

| 商业模式 | 售价 × 数量 | 耗材收入 |
|---------|-----------|---------|
| 一次性卖机器 | 999 × 1 = 999 | 0 |
| 机器 + 滤芯复购 | 999 × 1 + 60 × 2/年 × 3 年 = 1359 | +36% |

**这就是为什么智能硬件 PM 要在产品定义阶段考虑"耗材策略"：**

1. 滤芯尺寸和安装方式要**非标**（不能兼容公版滤芯）
2. 滤芯寿命检测（通过累计风扇运行时间或风压检测）会触发 App 推送换芯提醒
3. 机器本身售价可以压低，靠滤芯复购盈利

**PM 学到的东西：** 硬件的盈利模型不只有"卖一次"——耗材、订阅服务（云存储、空气质量报告）、配件，这些可能是比机器本身更稳定的收入来源。

### 踩坑复盘

**坑 1：PM2.5 读数不准**
- 问题：净化器显示 PM2.5 35μg/m³，但旁边放一个专业检测仪显示 50μg/m³
- 原因：激光传感器内部灰尘积累导致光路偏移
- 解决：在固件里加自动校准算法——通过风扇停转时的基线值校正

**PM 学到的东西：** 传感器会漂移。所有传感器都需要校准和校准补偿。这是硬件和软件最大的区别之一——软件的输出不会因为"用久了"变差，但硬件会。

**坑 2：自动模式不稳定**
- 问题：开启"自动模式"后，风扇忽快忽慢，用户投诉"神经质"
- 原因：PM2.5 传感器每 3 秒输出一个值，波动 ±5μg/m³。直接拿这个值控制风扇转速，导致风扇频繁调速
- 解决：在固件里加滑动平均滤波（取过去 30 秒的平均值）+ 转速变化死区（偏差 > 20μg/m³ 才调）

**PM 学到的东西：** 用户的感知不等于传感器的读数。传感器读数波动是正常的，但产品表现出来的行为必须平滑。这是嵌入式固件的价值所在——把"裸数据"变成"好体验"。

## 案例四：AI 智能摄像头

### 产品定位

- 产品：AI 人形检测智能摄像头，带云存储
- 目标售价：199-299 元
- 目标用户：家庭安防、宠物监控、老人看护

### 方案选型

```
┌─────────────────────────────────────────────────┐
│             主控 SoC（海思 Hi3516EV300）          │
│                    Linux 系统                     │
│  ┌──────────┐ ┌────────┐ ┌──────────┐           │
│  │ CMOS 传感器│ │ Wi-Fi │ │ NPU(神经网络)│          │
│  │ (SC2006) │ │ 模组   │ │  人形检测  │           │
│  └──────────┘ └────────┘ └──────────┘           │
└─────────────────────────────────────────────────┘
```

**和前面案例最大的区别：这用的是 MPU（微处理器），不是 MCU。**

因为摄像头需要处理：
- 视频编码（H.264/H.265 压缩）
- 网络协议栈（RTSP/HTTP/ONVIF）
- AI 推理（人形/宠物/移动检测）
- 云平台对接（MQTT/P2P 连接）

这些任务在 MCU 上无法完成。一台智能摄像头其实就是一台**小型 Linux 计算机**。

### BOM 拆解

| 部件 | 说明 | 单价 |
|------|------|------|
| SoC | Hi3516EV300（含 256MB DDR）| 45 元 |
| CMOS 传感器 | SC2006（2MP）| 15 元 |
| 镜头 | F1.6 3.6mm | 8 元 |
| WiFi 模组 | RTL8188（USB 接口）| 15 元 |
| Flash | 16MB SPI NOR（存固件）| 5 元 |
| 红外 LED 板 | 850nm 红外灯 + 光敏电阻 | 5 元 |
| 电机 | 水平+垂直云台电机 | 12 元 |
| PCB | 4 层板 | 10 元 |
| 电源 | 5V/2A 适配器 | 8 元 |
| 外壳 | 注塑 ABS | 12 元 |
| 包装 | 说明书+支架+螺丝 | 5 元 |
| **BOM 合计** | | **~140 元** |
| **零售价** | | **~199-299 元** |

**PM 要注意：** 智能摄像头的竞争极其激烈，BOM 140 元的产品零售价 199 元，利润空间极薄。盈利主要靠：
1. 云存储订阅（7 天回放 ~10 元/月）
2. 批量出货降低 BOM 成本
3. 多摄像头套装（单价降但总价升）

### 设计决策：人形检测在端侧还是云侧

| | 端侧（设备本地）| 云侧（服务器）|
|------|-------------|-------------|
| 硬件要求 | SoC 需要 NPU 或 DSP | 普通芯片就行，视频流上传 |
| 带宽 | 只上传告警截图（几 KB）| 持续上传视频流（几 Mbps）|
| 延迟 | 实时 | 取决于网络 |
| 功耗 | 较高（NPU 工作）| 较低（只做编码+上传）|
| 隐私 | 数据不上云 | 视频传到服务器处理 |
| 成本 | 芯片贵 15-30 元 | 云服务费 ~5 元/月/设备 |

**2019-2021 年主流是云侧：** 芯片便宜，上传带宽成本低，AI 能力集中部署。
**2022 年后主流转向端侧：** 海思/瑞芯微/Rockchip 推出了带 NPU 的 SoC，端侧人形检测的成本降到可以接受的程度。而且隐私法规（GDPR/PIPL）越来越严格，用户也更倾向"不上传视频"。

**PM 学到的东西：** 技术趋势会改变产品架构。2019 年做端侧 AI 是"不切实际"，2023 年做云侧 AI 是"不跟趋势"。产品定义阶段就要想清楚未来 2-3 年的技术路线。

### 踩坑复盘

**坑 1：红外灯微光下的噪点**
- 问题：夜间模式（红外灯开启）下画面噪点严重，用户投诉"看不清"
- 原因：降低成本用了便宜的 CMOS 传感器（SC2006），低照度性能差
- 解决方案：固件里做 3D 降噪（消耗 NPU 资源），或者换更好的传感器（+10 元成本）

**PM 学到的东西：** 参数表上写的"夜视 10 米"和有真实照度的测试结果是两回事。如果"夜视效果"是产品的核心卖点，不要在传感器上省钱。**在核心体验上省钱 = 给竞品送用户。**

**坑 2：云台回差**
- 问题：App 远程操作云台转动后，画面回不到精确的原始位置
- 原因：步进电机驱动齿轮有机械间隙（回差），步进数无法精确定位角度
- 解决：云台加装磁编码器（霍尔传感器），通过反馈闭环控制

**PM 学到的东西：** 云台"精确复位"是用户不常提但一旦没有就很恼火的功能。**机械结构的精度限制是硬件独有的问题**——软件里加一个变量就能解决的事情，在硬件里要加一个传感器、几根线、多一个算法模块。

## 总结：从四个案例中学到的

### 技术选型的趋势线

```
智能插座（2018）   双芯片 MCU+WiFi     BOM ~25元    零售 ~59元
智能插座（2022）   单芯片 ESP32-C3     BOM ~18元    零售 ~49元

门锁（2019）     Nordic nRF52        BOM ~200元   零售 ~699元
```

**PM 要记住的趋势：** 国产芯片替代 + 集成度提高 = 每 2-3 年 BOM 成本降 20-30%。

### 硬件 PM 最常见的认知偏差

| 软件出身的 PM 通常会认为 | 硬件现实 |
|------------------------|---------|
| "这个功能只是加几行代码" | 可能涉及换芯片/改 PCB/重做认证 |
| "第一版就能稳定" | 至少 2-3 次打板迭代才能到量产品质 |
| "性能看参数表就够了" | 参数表测的都是实验室理想条件 |
| "用户需求不变" | 用户需求会变，但硬件变了就是几周 |
| "供应商会给最便宜的" | 供应商给的是他们利润率最高的方案，不一定是最适合你的 |
| "认证可以跟开发并行" | 认证必须等硬件定型后才能做，而且第一次很可能不过 |

### 硬件 PM 应该建立的三个习惯

1. **拿到一个新产品的第一反应是拆 BOM** —— 这个东西多少钱做出来的？哪里最贵？这个价格能否支持目标售价？
2. **看到任何技术方案的第一反应是"它的妥协在哪"** —— 没有完美的方案，每个选择背后都有取舍。知道妥协在哪，才知道风险在哪。
3. **每次踩坑都要记录下来** —— 硬件产品的坑有极强的复用性。这个项目踩过的坑，下个项目大概率还会踩。记录下来就是团队的技术资产。
