# 电路基础速成

> 写给软件转行者的电子学入门 —— 不讲公式推导，只讲你作为 PM/嵌入式工程师需要理解的核心概念。

---

## 前言：这课是给谁看的

你写代码的时候，电压和电流是抽象的"高电平""低电平"。但在硬件世界里，**电平不是抽象的——它是真实的 0V 或 3.3V**。不理解电路基础，你在做这些决策时会完全处于模糊状态：

- 为什么 I2C 总线要加上拉电阻？（不加会怎样？）
- 为什么电池电压不能直接接 ADC 引脚？（会烧芯片）
- 为什么工程师说"这个地方要加一个 0.1μF 电容"？（不加也能跑，但会出奇怪的问题）
- 为什么同一个芯片 64 引脚版比 48 引脚版贵 10 元？（封装和散热）

这课的目标不是让你能设计电路，而是让你**能听懂硬件工程师在说什么、能看懂简单的原理图、能做出有理有据的决策**。

## 第一章：电压 · 电流 · 电阻 —— 电学三兄弟

### 1.1 最直观的理解：水管模型

```
电压 = 水压          （单位：V，伏特）
电流 = 水流量        （单位：A，安培）
电阻 = 水管粗细      （单位：Ω，欧姆）
```

- 水压（电压）越大，水（电流）流得越猛
- 水管越细（电阻越大），水（电流）流得越困难
- 水压 = 水流量 × 水管阻力（V = I × R）

**这就是欧姆定律，整个电子学的根基。**

### 1.2 在嵌入式世界里，这些数值通常是多少

| 量 | 常见值 | 感性理解 |
|----|-------|---------|
| 电压 | 3.3V / 5V（MCU 工作电压）| 一颗普通电池 1.5V |
| 电压 | 12V / 24V（工业设备）| 汽车点烟器 12V |
| 电流 | μA（微安，待机）| 10⁻⁶ 安培，极微小 |
| 电流 | mA（毫安，工作）| 10⁻³ 安培。一颗 LED 约 10mA |
| 电流 | A（安培，电机/加热）| 大功率设备 |
| 电阻 | 220Ω（LED 限流）| 粗导线几乎 0Ω，断开是无穷大 |
| 电阻 | 10kΩ（上拉/下拉）| 标准上拉电阻值 |
| 电阻 | 4.7kΩ（I2C 上拉）| I2C 总线标准 |

### 1.3 一个实际计算：为什么 LED 要串电阻

单片机 GPIO 输出 3.3V。LED 正向压降约 2V，工作电流约 10mA。

如果不串电阻：3.3V 直接加到 LED 上 → 电流过大 → LED 烧毁。

串电阻后：R = (3.3 - 2) / 0.01 = 130Ω。实际取 220Ω（常见电阻值，留余量）。

### 1.4 功率：当心 "它扛不扛得住"

功率 = 电压 × 电流（单位：W，瓦特）

- 一个 LED 功率：3.3V × 0.01A = 0.033W（极微小）
- 一个电机功率：12V × 2A = 24W（开始发热了）
- 一个加热器功率：220V × 10A = 2200W（很热，需要散热）

**PM 要注意：** 功率 = 发热。需要大功率的产品（电机、加热、大屏背光）必须考虑散热。散热孔、风扇、散热片——这些不是可选的，是必要的。

## 第二章：常见元器件（PM 版）

### 2.1 电阻

| 特性 | 解释 |
|------|------|
| 作用 | 限制电流、分压、上拉/下拉 |
| 符号 | 原理图上画成一个锯齿状 |
| 单位 | Ω（欧姆）、kΩ（千欧）、MΩ（兆欧）|
| 选型参数 | 阻值和功率（1/8W、1/4W 等）|
| PM 要知道的 | 单片机电路里最常用的值是 220Ω、1kΩ、4.7kΩ、10kΩ |

**在嵌入式里，电阻最常见的三个用途：**

1. **限流（保护 LED / 晶体管基极）** —— 防止电流过大烧坏器件
2. **上拉/下拉（让引脚在不被驱动时有确定电平）** —— 按键、I2C 总线、复位引脚
3. **分压（把高电压降到 ADC 能测的范围）** —— 测量电池电压

### 2.2 电容

| 特性 | 解释 |
|------|------|
| 作用 | 储存电荷、滤波、去耦 |
| 符号 | 两根平行线（电解电容画 + - 号）|
| 单位 | F（法拉），常用 μF（微法）、nF（纳法）、pF（皮法）|
| PM 要知道的 | 0.1μF = 100nF，这是 PCB 上最常用的去耦电容值 |

**电容在嵌入式里的三件事：**

1. **去耦电容（0.1μF）** —— 每个 IC 芯片的电源引脚旁必须放一个。它的作用是"稳压"：当芯片瞬间需要大电流时，电容就近放电补充，不让电压跌落。**不加的话，MCU 可能会莫名其妙地复位、死机、通信出错。** 这不是玄学，是物理。
2. **滤波电容（10μF ~ 100μF）** —— 放在电源入口，滤除低频纹波
3. **延时/定时** —— 电容充放电需要时间，可以用来做简单的延时、按键消抖

### 2.3 二极管

| 特性 | 解释 |
|------|------|
| 作用 | 只允许电流单向流动 |
| 符号 | 一个三角形 + 一条竖线 |
| 关键参数 | 正向压降（通常 0.7V）|

**在嵌入式里的应用：**
- **保护电路**：在电源入口反接一个二极管，电源接反时不会烧电路
- **续流二极管**：在继电器/电机线圈旁并联，防止断电时的反向高压击穿驱动管
- **LED**：发光二极管，本质就是一种二极管

### 2.4 三极管 vs MOSFET（PM 需要知道的区别）

| | 三极管（BJT） | MOSFET |
|------|------------|--------|
| 控制方式 | 电流控制 | 电压控制 |
| 消耗 | 基极有电流消耗 | 栅极只需电压，几乎不耗电 |
| 开关速度 | 较慢 | 较快 |
| 嵌入式常用场景 | 小电流开关、信号放大 | 大电流开关（电机/LED 驱动） |

**一句话区分：** MOSFET 比三极管更省电、速度更快，但成本略高。在电池供电的 IoT 设备里，**优先用 MOSFET**。

### 2.5 PM 需要认识的几个封装名字

封装 = 元器件外面那个塑料壳子的形状。

| 封装名 | 大小约 | 焊接方式 |
|--------|-------|---------|
| THT（直插）| 几毫米到几厘米 | 手焊/波峰焊 |
| SOT-23 | 3×1.5mm | 机器贴片 |
| SOIC-8 | 5×4mm | 机器贴片 |
| QFN | 3×3mm ~ 7×7mm | 机器贴片（底部有焊盘）|
| BGA | 焊球在底部 | 需要 X 光检查 |

**PM 要知道：** 封装越小越便宜，但焊接和维修越难。BGA 封装一旦虚焊，X 光都未必看得出，维修成本很高。物联网产品尽量避免 BGA，除非真的需要那个性能。

## 第三章：看原理图

### 3.1 原理图长什么样

一个典型的 MCU 原理图页面包括：

```
┌─────────────────────────────────────────────┐
│  电源部分                                     │
│  5V ──[LDO]── 3.3V ───★── 10μF ── GND       │
│                        ★── 0.1μF ── GND      │
├─────────────────────────────────────────────┤
│  MCU 核心                                     │
│  ┌─────────────────────┐                     │
│  │                     │                     │
│  │  STM32F103          │─── PA0 ───→ 传感器  │
│  │                     │─── PA1 ───→ 按键     │
│  │                     │─── PA9 (TX) ──→ 蓝牙 │
│  │                     │─── PA10(RX) ←── 蓝牙 │
│  └─────────────────────┘                     │
├─────────────────────────────────────────────┤
│  外设接口                                     │
│  I2C: PB6(SCL) ──[4.7kΩ]── 3.3V             │
│       PB7(SDA) ──[4.7kΩ]── 3.3V              │
│                                              │
│  UART: PA9(TX) ──→ 蓝牙模块 RX               │
│        PA10(RX) ←── 蓝牙模块 TX              │
└─────────────────────────────────────────────┘
```

### 3.2 看原理图时 PM 要抓住的重点

1. **电源树**（Power Tree）—— 供电是怎么走的。5V → LDO → 3.3V → 各模块。这个决定你的电池/电源选型
2. **接口编号**—— 哪个外设接了哪个引脚。这决定了引脚够不够用
3. **上拉/下拉电阻**—— I2C 总线上有没有 4.7kΩ 上拉？复位引脚有没有 10kΩ 上拉？这些漏了会影响通信可靠性
4. **去耦电容**—— 每个 IC 的电源脚旁边有没有 0.1μF 电容？没有的话是偷工减料
5. **接插件定义**—— USB 座、电源座、传感器接口的引脚定义。这个决定了你的线束怎么接

### 3.3 原理图上常见的网络标号

| 标号 | 含义 | PM 要知道 |
|------|------|----------|
| VCC / VDD | 正电源（3.3V 或 5V）| 芯片供电 |
| GND / VSS | 地（0V 参考点）| 所有电压的参考基准 |
| VREF | 参考电压（ADC 校准用）| 影响 ADC 精度 |
| VBAT | 电池供电引脚 | 给 RTC 供电 |
| nRST | 复位引脚（低电平复位）| 通常接上拉电阻 + 按键 |
| BOOT0/1 | 启动模式选择 | 决定了上电后从 Flash 还是系统存储器启动 |
| NC | 未连接 | 这个引脚悬空 |

## 第四章：嵌入式里的几个重要电路概念

### 4.1 什么是上拉/下拉电阻

**问题：** MCU 的输入引脚在悬空（没有被外部电路驱动）时，电平是不确定的。它可能读到高、可能读到低、可能读到 1MHz 的噪声。

**解决方案：** 用一个电阻（通常 10kΩ）把引脚拉到确定的电平。

- **上拉**：电阻接到 VCC（3.3V）。不按时引脚读到高电平。**按键常用**：按下接地，引脚变低。
- **下拉**：电阻接到 GND。不按时引脚读到低电平。

**I2C 总线的特殊情况：** I2C 的 SCL 和 SDA 需要 4.7kΩ 上拉电阻。因为 I2C 协议要求设备只能拉低总线，不能拉高——高电平是靠上拉电阻"被动上拉"的。**不加这个电阻，I2C 通信会间歇性失败。**

### 4.2 什么是分压（为什么电池不能直接接 ADC）

ADC 只能测量 0~3.3V 的电压。锂电池充满 4.2V，直接接会烧 ADC。

**解决方案：** 分压电路

```
电池 4.2V ──[R1 10kΩ]──★──[R2 10kΩ]── GND
                      │
                   ADC 输入（2.1V）
```

Vout = Vin × R2 / (R1 + R2) = 4.2 × 10 / 20 = 2.1V

这个 2.1V 在 ADC 的安全范围内。然后在固件里把 ADC 值乘以 2 还原成真实电压。

**PM 要知道：** 分压电路是嵌入式里**测量模拟量**的基础。温度（NTC 热敏电阻 + 固定电阻分压）、电池电压、电位器——全是用分压实现的。

### 4.3 什么是开漏输出

开漏（Open-Drain）：输出端内部只连接了 N-MOSFET 的"漏极"到 GND，不连接 VCC。

```
VCC ──[上拉电阻]──★── IO 引脚（到外部）
                   │
                  MOS
                   │
                  GND
```

- 当 MOS 导通时：引脚拉到 GND（输出低电平）
- 当 MOS 截止时：引脚通过上拉电阻拉到 VCC（输出高电平）

**用处：** I2C 总线、多个设备共用一个中断线。开漏允许多个输出"线或"在一起——谁拉低谁就输出低，不会打架。

**PM 要知道：** 当工程师说"这个引脚是开漏的"，意思是它需要外接上拉电阻才能输出高电平。如果原理图上忘记画上拉电阻，这个引脚的通信会一直失败。

### 4.4 什么是施密特触发器

MCU 的 GPIO 输入引脚都内置施密特触发器。它的作用是**消除噪声带来的抖动**：

```
输入信号（有噪声）：
  ╱╲╱╲╱╲╱╲╱╲╱╲
  ╱╱╲╲╱╱╲╲╱╱╲╲

经过施密特触发器后：
  ────    ────    ────
      ────    ────
```

施密特触发器有两个阈值：从低到高要超过阈值 Vth（比如 2.0V），从高到低要低于阈值 Vtl（比如 0.8V）。这个"磁滞"特性防止了信号在阈值附近来回跳变。

**PM 要知道：** 施密特触发器让 MCU 在有噪声的环境下也能稳定读取电平。但如果噪声太大（比如电机启动时的 EMI），还是需要外部消抖电路（RC 滤波器）。

## 第五章：电源

### 5.1 LDO —— 低压差线性稳压器

LDO 是最常用的电源转换方案。输入一个较高电压，输出一个稳定的较低电压。

```
5V ──→ [LDO] ──→ 3.3V
```

**优点：** 简单、便宜、噪声低、只需要输入输出两个电容
**缺点：** 效率低（效率 = Vout/Vin = 3.3/5 = 66%），多余的电压以热量散掉

**PM 要知道：** 如果电池供电，LDO 会把多余的电压烧成热量——这就是工程师说的"LDO 效率低"。电池电压从 4.2V 降到 3.3V，24% 的电被白白浪费。如果要长续航，用 DCDC。

### 5.2 DCDC —— 开关电源

Buck（降压）、Boost（升压）、Buck-Boost（升降压）

```
                     ┌─ 电感 ─┐
Vin ──→ [开关 MOS]──★──★──→ Vout
                    │   │
                   二极管 电容
                    │   │
                   GND  GND
```

**优点：** 效率高（85-95%），可以升压也可以降压
**缺点：** 电路复杂、输出有纹波、成本略高、需要 PCB Layout 更小心

**PM 要知道：** DCDC 的效率远远好于 LDO，但 DCDC 的开关噪声会影响射频（WiFi/BLE 灵敏度）。所以很多产品是 DCDC 先把电池电压降到 3.3V 附近，再用 LDO 二次稳压给射频模块供电——兼顾效率和噪声。这就是 **两级供电** 的常见做法。

### 5.3 常见电源需求（PM 用来评估工程师给的数据）

| 芯片/模块 | 电压 | 典型电流 | 供电方案 |
|----------|------|---------|---------|
| MCU (STM32) | 3.3V | 50-200mA | LDO 或 DCDC |
| ESP32 (WiFi) | 3.3V | 峰值 500mA | DCDC（电流太大，LDO 会过热）|
| BLE 模块 | 3.3V | 10-50mA | LDO |
| LED x 1 | 3.3V | 10mA | GPIO 直接驱动（串电阻）|
| 步进电机 | 5-12V | 500mA-2A | 驱动芯片 + DCDC |
| 舵机 | 5-6V | 500mA | 单独 DCDC |
| 锂电池（单节） | 3.7-4.2V | — | 需要充电管理芯片（TP4056 等）|

## 第六章：数字电平与接口

### 6.1 逻辑电平标准

| 标准 | VCC | 低电平阈值 | 高电平阈值 | 兼容性 |
|------|-----|-----------|-----------|-------|
| 5V TTL | 5V | <0.8V | >2.0V | 最老的标准 |
| 3.3V LVTTL | 3.3V | <0.8V | >2.0V | 大部分 MCU |
| 1.8V | 1.8V | <0.6V | >1.2V | 新芯片、FPGA |

**电平不匹配的例子：** 一个 5V 的传感器直接接到 3.3V 的 MCU 引脚。传感器输出的高电平是 5V，而 MCU 引脚的耐压上限是 3.6V → 烧 MCU。

**解决方案：**
- 电平转换器（分立 MOSFET 或集成芯片）
- 分压电阻
- 选相同电平的器件

**PM 要知道：** 电平不匹配是硬件联调阶段最常见的烧芯片原因之一。选型时确认所有 IC 的供电电压一致。

### 6.2 I2C 为什么需要上拉电阻

I2C 是开漏总线。每个设备只能拉低总线，不能拉高。高电平是靠**上拉电阻**提供的。

```
SCL ──[4.7kΩ]── 3.3V
SDA ──[4.7kΩ]── 3.3V
```

如果总线上有多个设备，任何一个设备拉低 SCL/SDA，对应的线就被拉低。没有设备拉低时，上拉电阻把线拉回高电平。

**上拉电阻的值很重要：**
- 太大（>10kΩ）：上升沿太慢，高频通信时波形跟不上
- 太小（<1kΩ）：设备拉低时需要灌入的电流太大，可能损坏器件

### 6.3 什么是灌电流和拉电流

| 方向 | 含义 | 典型值 |
|------|------|-------|
| 拉电流（Source）| MCU 引脚向外输出电流（驱动 LED）| 通常 4-8mA |
| 灌电流（Sink）| 外部电流流入 MCU 引脚 | 通常 4-8mA |

**PM 要知道：** MCU 的 GPIO 引脚能驱动的电流很有限（几 mA）。要驱动大负载（继电器、电机、大功率 LED），必须加**驱动芯片**或**晶体管/MOSFET**。直接用 GPIO 驱动大负载，会烧 MCU。

## 第七章：PCB 基础

### 7.1 PCB 的层数

| 层数 | 复杂度 | 价格对比（同面积）| 适用场景 |
|------|-------|----------------|---------|
| 2 层 | 低 | 基础价 | 简单产品、低频电路、样板 |
| 4 层 | 中 | 2 层板的 2-3 倍 | 大多数物联网产品——信号层 + 地层 + 电源层 + 信号层 |
| 6 层+ | 高 | 4 层板的 2-3 倍 | 高速信号、射频、BGA 芯片 |

**PM 要知道：** 4 层板比 2 层板贵不止一倍，但电磁兼容性（EMC）好很多。如果产品需要通过 FCC/CE 认证，2 层板通过难度大很多——很可能到了认证阶段才发现不过，被迫改 4 层板，浪费时间和打板费。**建议直接上 4 层板。**

### 7.2 打板费用参考

| 项目 | 2 层样板（10 片）| 4 层样板（10 片）| 批量（1000 片）|
|------|---------------|---------------|--------------|
| 打板费 | ~300 元 | ~1000 元 | ~2-5 元/pcs |
| 焊接费（样板手焊）| ~200 元 | ~400 元 | ~0.5-2 元/pcs |
| 周期 | 5-7 天 | 7-10 天 | 15-20 天 |

**PM 要知道：** 打板 + 焊接的周期不可压缩。每次改版至少 2 周。**不要指望一次打板成功**——至少预留 2-3 次打板的预算和时间。

### 7.3 什么是 Layout 上的关键"陷阱"

1. **天线下面不能走地线和电源线** —— WiFi/BLE 天线需要干净的"净空区"。天线下方如果有铜箔，会吸收辐射能量，导致通信距离缩短 50% 以上
2. **大电流路径要加宽走线** —— 1mm 宽的铜箔大约只能通过 1A 电流。走大电流的线太细 = 发热 = 铜箔烧断
3. **模拟信号远离数字信号** —— ADC 测量线路紧挨着 SPI 高速时钟信号线 → ADC 读数抖动
4. **电源入口先过电容再过芯片** —— 先给去耦电容再给芯片供电，而不是先到芯片再到电容

## 第八章：测试工具

### 8.1 万用表

PM 不需要会用，但要知道它能测什么：

| 功能 | 做什么 | 在嵌入式里用来 |
|------|-------|-------------|
| 电压测量 | 测两点之间的电压差 | 测电池有没有电、LDO 输出是不是 3.3V |
| 通断测试（蜂鸣档）| 测两根线是否导通 | 检查焊接有没有虚焊、线有没有断 |
| 电阻测量 | 测电阻值 | 检查电阻有没有焊对 |
| 二极管测试 | 测二极管单向导通性 | 检查二极管有没有焊反 |

### 8.2 示波器

- 价格：几百到几万
- 用途：**看波形**。看信号有没有毛刺、上升沿够不够陡、PWM 占空比对不对、通信协议有没有信号完整性
- PM 要知道：这是硬件工程师的主战工具。当工程师说"要抓一下波形"，意思是要用示波器调试信号质量问题

### 8.3 逻辑分析仪

- 价格：几十到几百（Saleae 克隆版约 30 元）
- 用途：**看数字信号**。解码 UART / I2C / SPI 的通信数据
- PM 要知道：这是嵌入式工程师调通信协议的主战工具。当 I2C 读不到传感器数据时，逻辑分析仪一看就知道"SCL 时钟在走，SDA 上没有任何 ACK"——定位到是传感器没响应

### 8.4 什么情况下需要什么工具

| 场景 | 用什么工具 |
|------|----------|
| 板子完全不工作，检查电源有没有上 | 万用表（测电压）|
| LED 不亮，检查引脚有没有输出 | 万用表（测引脚电平）|
| UART 打印乱码，调整波特率 | 逻辑分析仪（抓一帧看每位的宽度）|
| I2C 读不到设备，检查有没有 ACK | 逻辑分析仪（看 SCL/SDA 时序）|
| PWM 呼吸灯不平滑，检查波形 | 示波器（看 PWM 波形的上升/下降沿）|
| WiFi 距离太短，检查天线匹配 | 频谱仪 / 网络分析仪（专业测试）|

## 附录 A：常见电路"黑话"翻译

| 工程师说 | 翻译成 PM 能懂的话 |
|---------|-----------------|
| "这个地方浮空了" | 引脚没接任何东西，电平不确定 |
| "信号有毛刺" | 波形上有噪声，可能会误触发中断 |
| "上下沿不够陡" | 信号从低到高变化太慢，通信可能出错 |
| "电源纹波太大" | 电源输出有波动，可能影响传感器精度 |
| "地弹" | 大电流瞬间通过地线时，地线电压被抬升了——会导致所有电平参考偏移 |
| "串扰" | 一根信号线上的能量耦合到了相邻的信号线上 |
| "阻抗不匹配" | 信号在传输线上反射，导致波形变形 |
| "热设计没做好" | 大功率器件散热不够，会过热关机 |
| "共地问题" | 两个电路的地线没有连好，电平参考点不一样 |
| "电源上电顺序不对" | 某些芯片需要先上内核电压再上 I/O 电压，搞反了可能锁死芯片 |

## 附录 B：选型时的电源计算速查

**产品续航估算：**

```
续航时间（小时）= 电池容量（mAh）÷ 平均电流（mA）

例：1000mAh 电池，设备平均电流 50mA
续航 = 1000 ÷ 50 = 20 小时

如果要续航 1 年（8760 小时）：
平均电流 ≤ 1000 ÷ 8760 ≈ 0.114mA = 114μA
```

**产品功耗分配（典型 IoT 设备）：**

| 模块 | 工作电流 | 待机电流 | 占空比 |
|------|---------|---------|-------|
| MCU | 10mA | 5μA | 99.9% 待机 |
| WiFi 模块 | 300mA | 10μA | 0.1% 时间在发数据 |
| 传感器 | 1mA | 1μA | 1% 时间在读 |
| LED 指示灯 | 10mA | 0 | 0.1% 时间在闪 |

**PM 能做的最有效的续航优化：** 不是换更大电池，而是**降低待机电流 + 缩短工作占空比**。把 WiFi 发送时间从 1 秒缩短到 100ms，续航可以提升 10 倍。比把电池从 1000mAh 换成 2000mAh 有效得多——而且不增加 BOM 成本和产品体积。
