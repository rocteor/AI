# PRD：AI 智能家庭安防摄像头

> 产品版本：V1.0 · 2026年7月
> 这是一份演示性质的 PRD，展示如何将 AI 硬件知识清单转化为完整的产品定义。

---

## 1. 产品概述

### 1.1 产品定位

AI 人形检测智能摄像头，主打**隐私保护 + 离线可用 + 无需月费**的家庭安防产品。

### 1.2 目标售价

- 零售价：229 元（首发价）、299 元（标准价）
- 目标 BOM：≤ 140 元
- 毛利率目标：≥ 35%（扣除研发、认证、渠道后）

### 1.3 目标用户

| 用户画像 | 需求 | 愿意为此付费吗 |
|---------|------|--------------|
| 有小孩的家庭 | 看护宝宝，想知道宝宝醒了没 | ✅ 是 |
| 养宠物的年轻人 | 上班时看猫狗在干嘛 | ✅ 是 |
| 独居女性 | 安全监控，有人闯入立即告警 | ✅ 是 |
| 老人看护 | 远程看老人是否摔倒/异常 | ✅ 是 |
| 对隐私敏感的用户 | 不想让视频上传到任何服务器 | ✅ **这是核心卖点** |

### 1.4 核心卖点（一句话）

> **"不上传视频的 AI 摄像头——本地识别、实时告警、隐私无忧。"**

## 2. 竞品分析

### 2.1 市场格局

| 产品 | 价格 | AI 方式 | 月费 | 隐私 |
|------|------|--------|------|------|
| 小米智能摄像头 2K | 199 元 | 云侧 AI 检测 | 云存储月费 | ❌ 上传云端 |
| 萤石 C6WI | 269 元 | 云侧 AI 检测 | 云存储月费 | ❌ 上传云端 |
| TP-Link Tapo C225 | 229 元 | 云侧 AI 检测 | 无月费 | ⚠️ 本地 SD 卡 |
| **本品（AI 端侧）** | **229-299 元** | **端侧 NPU 检测** | **0 元** | **✅ 不上传** |

### 2.2 差异化定位

市面上 199-299 元的摄像头都走"云侧 AI"路线——视频上传到云端做人形检测。本品的差异化在于：

1. **端侧 NPU 本地检测** —— 视频不上传，隐私是卖点不是担忧
2. **断网可用** —— 即使家里 WiFi 断了，人形检测和 SD 卡录制不受影响
3. **零月费** —— 不需要为 AI 功能支付订阅费
4. **低延迟** —— 本地检测响应 < 100ms，云侧方案通常 1-3 秒

### 2.3 竞争风险

- 如果竞品跟进端侧 AI 方案并打价格战，溢价空间会缩小
- 时间窗口：预计 2026-2027 年是端侧 AI 摄像头从"差异化"到"标配"的过渡期

## 3. 功能清单（按优先级）

### P0（必须有，MVP 必须包含）

- [ ] 1080P 实时视频预览（手机 App 端）
- [ ] 本地 AI 人形检测（端侧 NPU 推理，不上传视频）
- [ ] 人形检测触发时：App 推送告警通知
- [ ] 人形检测触发时：SD 卡录制 10 秒短视频
- [ ] 告警截图上传到云端（仅截图，非视频流）
- [ ] 水平+垂直云台控制（App 端远程操作）
- [ ] 双向语音（对讲）
- [ ] 红外夜视（自动切换）
- [ ] WiFi 配网（BLE 辅助配网）

### P1（V1.1 迭代添加）

- [ ] 宠物检测模式（区分人形和宠物，减少误报）
- [ ] 移动检测区域划定（用户在画面中画一个区域）
- [ ] 自动驾驶追踪（检测到人形后自动跟随转动）
- [ ] 隐私遮蔽模式（手动遮挡镜头）
- [ ] 告警时段设置（例如 23:00-7:00 开启，白天关闭）
- [ ] 多摄像头同屏查看

### P2（V2.0 考虑）

- [ ] 人脸识别（区分家人和陌生人）
- [ ] AI 哭声检测（婴儿哭时告警）
- [ ] AI 跌倒检测（老人摔倒告警）
- [ ] 云存储 7 天回放（付费订阅）

### 3.1 AI 功能的端侧/云侧分配决策

| 功能 | 处理位置 | 决策理由 |
|------|---------|---------|
| 人形检测 | ✅ 端侧 NPU | 需实时（<100ms），隐私重要，断网可用 |
| 宠物检测 | ✅ 端侧 NPU | 同上，和人形检测共享同一个模型 |
| 人脸识别 | ⚠️ V1 不做，V2 做端侧 | 端侧 NPU 算力够，但开发周期长 |
| 告警推送 | 端侧触发 → 云侧推送 | 推送必须有网络，适合云侧 |
| 截图云存储 | 端侧上传到云 | 截图体积小（几十 KB），带宽成本低 |
| 视频回放 | SD 卡本地 | 视频流太大，上传云端成本高（月费模型）|

## 4. 技术方案

### 4.1 芯片选型决策

**候选方案对比：**

| 芯片 | NPU 算力 | 价格 | 开发难度 | 选型评估 |
|------|---------|------|---------|---------|
| 海思 Hi3516EV300 | 0.5 TOPS | ~45 元 | 中（Linux 平台）| ⚠️ 供货不稳定 |
| **瑞芯微 RV1106** | **0.5 TOPS** | **~30 元** | **中（Linux + RKNN）** | **✅ 选型推荐** |
| 算能 CV1821 | 1 TOPS | ~25 元 | 高（SDK 不成熟）| ❌ 开发风险大 |
| 全志 V833 | 1 TOPS | ~35 元 | 中 | ⚠️ 社区支持少 |

**最终选型：瑞芯微 RV1106**

**理由：**
1. **性价比最优** —— 30 元价位带 NPU 0.5 TOPS，人形检测够用
2. **瑞芯微 AI 工具链成熟** —— RKNN Toolkit 支持 TensorFlow/PyTorch/ONNX 模型转换，落地快
3. **供货稳定** —— 瑞芯微是国产大厂，2023-2024 年未见严重缺货
4. **社区生态** —— 瑞芯微的 Linux SDK + NPU 文档比其他国产厂商好得多
5. **成本低 33%** —— 比海思 Hi3516 方案省 15 元，直接转化为价格竞争力

### 4.2 方案架构

```
┌──────────────────────────────────────────────────────────┐
│              瑞芯微 RV1106 SoC                            │
│                  Linux 4.19 / Buildroot                   │
│                                                          │
│  ┌──────────────┐  ┌────────────┐  ┌──────────────────┐ │
│  │ CMOS 传感器  │  │  NPU       │  │  WiFi (RTL8188)  │ │
│  │ SC2232 (2MP) │  │  人形检测  │  │  USB 接口        │ │
│  └──────┬───────┘  └────────────┘  └──────────────────┘ │
│         │                                         │      │
│    MIPI CSI 接口                               USB 总线 │
└─────────┼──────────────────────────────────────────┼──────┘
          │                                          │
     ┌────▼──────┐                          ┌───────▼──────┐
     │ 25F47S   │                          │ SD 卡 / 云平台 │
     │ 红外 LED  │                          │ MQTT + P2P    │
     └──────────┘                          └───────────────┘
```

### 4.3 AI 模型方案

| 要素 | 方案 |
|------|------|
| 检测模型 | MobileNet-SSD 或 YOLOv5-nano（量化后 < 1MB）|
| 模型格式 | RKNN（瑞芯微 NPU 原生格式）|
| 训练框架 | TensorFlow 2.x / PyTorch（转 ONNX → RKNN）|
| 检测精度 | 目标 > 90%（室内场景），目标 > 85%（室外弱光）|
| 检测帧率 | ≥ 15 FPS（NPU 处理，不占用 CPU）|
| 模型更新 | OTA 推送新模型（通过 App → 云端 → 设备）|

## 6. 开发计划

### 6.1 时间线

```
阶段                      时间          PM 关键工作
────────────────────────────────────────────────────
P0 需求确认               W1-W2        出本 PRD，确认功能清单
芯片采购                   W2           下单 RV1106（交期 8-12 周）

打板                      W6-W7        打板费 ~2000 元（4 层）
样板焊接+Linux 移植        W8-W10       瑞芯微 SDK 适配
NPU 模型移植 + 调优        W11-W13      人形检测模型量化 + 精度验证
外设驱动（Sensor/WiFi）    W12-W14      调通视频采集+WiFi 联网
App 开发（Android/iOS）    W12-W16      可复用部分，与固件联调
整机联调                   W15-W17      所有功能调通
小批量 + 产测              W18-W19      编写产测脚本
认证（FCC/CE/SRRC）        W20-W23      4-6 周，2-5 万元
试产 + 爬坡                W24-W26      跟踪良率
首发                       W27          备货 5000 台
```

### 6.2 关键风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| RV1106 芯片交期延误 | 中 | 整个项目延期 | 提前下单，备选方案：海思 Hi3516EV300（新 MCU 需 +2 周适配）|
| NPU 模型精度不达标 | 中 | 人形检测误报率高，用户投诉 | 预留模型迭代时间（2 轮量化+验证），备选提升到 1 TOPS 芯片 |
| 夜视效果差（红外噪点）| 低 | 用户退货 | 选型阶段用 SC2232 而非最便宜的 SC2006，固件中加 3D 降噪 |
| 云台控制精度不够 | 中 | 操作体验差 | 云台加磁编码器做闭环控制，预留 2 周调试时间 |

## 7. PM 的 AI 功能验收标准

### 人形检测（核心 AI 功能）

| 验收项 | 合格标准 | 测试方法 |
|--------|---------|---------|
| 室内日间检测率 | ≥ 95% | 在室内正常光照下，测试 200 次人经过画面 |
| 室内夜间（红外）检测率 | ≥ 90% | 在完全黑暗环境下，测试 100 次人经过画面 |
| 误报率 | ≤ 5%（每 24 小时）| 在无人的房间内运行 48 小时，记录告警次数 |
| 检测延迟 | ≤ 500ms（从人到画面到告警推送）| 用秒表实测 |
| 宠物误报 | ≤ 10%（宠物移动触发告警）| 让猫/狗在画面内活动 1 小时 |
| 断网检测 | ✅ 断网后仍然能检测并录制到 SD 卡 | 拔掉网线测试 |

### 用户验收标准

- 首次配网时间 ≤ 2 分钟（从开机到正常使用）
- App 端预览延迟 ≤ 1 秒
- 云台转动响应 ≤ 300ms
- SD 卡回放流畅（≥ 15 FPS）
